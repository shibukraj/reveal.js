Platform Architecture

Notes

Platform Components

  • Data Store a.k.a Data Layer
    1. Oracle - ODP.Net
    2. Future - Data Store agnostic. Think Cloud!
  • Application Logic - a.k.a Business Layer
    1. ASP.Net MVC Web API 2.0
    2. Future - .Net Core 2.0 or above. Think Linux!
  • User Interaction - a.k.a User Interface Layer
    1. NodeJS 8.x (Terminal Server & UI)
    2. Future - iOS/Android. Think Mobile!
Ending Notes

Design Considerations

  • Each tier - designed & architected in isolation.
  • Distributed over http/https
  • MVC Pattren - UI
  • MVC & CQRS (Modified) Pattren - Business
  • N Tier - Logical & Physical
    1. Not a SPA
    2. Not a classic 3 Tier.
    Why these Considerations
  1. Well understood and documented.
  2. Proven to be effective; but not fool proof.
  3. Simple & well known protocols for distribution.
  4. Each tier - designed & architected in isolation.
    1. Will help to utilize the best of each tier.
