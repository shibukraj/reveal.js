Why a new platform?

  1. Lower time to market
  2. Reliable, Scalable and Resilient
  3. Aligns with OHL disaster strategy and principles
  4. Modern/newer - architecture, systems design and programming language

Lower time to market

  1. Message driven vs customer/implemenation driven
  2. Follows configuration over construction
  3. Mock host servers. Less dependency with vendors
  4. Unit test as code
    • Repeatable/verifiable for every changes
    • Automated and Immediate feedback

Reliable, Scalable and Resilient

  1. Message Oriented. No database triggers
    • Drop in new messages!
    • Ability to reconfigure messages
  2. Disaster recovery and improved resilience via load balancers

Architecture

  1. Cloud centric architecture
    • Ability to scale using commodity hardware
    • Run in cloud - if needed
  2. Separation of concerns and single responsibility per message
  3. Linear chain of command
